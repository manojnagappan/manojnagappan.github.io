Q-LEARNING VEHICLE BATTLE

This simulation demonstrates a reinforcement learning system where the RED vehicle learns to chase and attack the GREEN player using Q-learning.

OBJECTIVE

  • RED vehicle = AI agent.
  • GREEN vehicle = human player.
  • If AI shoots player before timer ends → AI wins.
  • If player survives until timer reaches zero → Player wins.

PLAYER CONTROLS

  • ↑ : Accelerate
  • ↓ : Reverse
  • ← : Rotate Left
  • → : Rotate Right

HOW THE AI LEARNS

The AI uses a Q-table where each state stores values for multiple actions.

The state is based on:

  • Relative X direction
  • Relative Y direction
  • Distance bucket

The agent learns which actions are best for each state using rewards and penalties.

Q-TABLE ANALYTICS

  • STATE INDEX axis = all learned state-action entries.
  • Q VALUE axis = learned reward score.
  • TOTAL STATES = number of unique environment states learned.
  • TOTAL ACTIONS = actions available to the AI.
  • MAX Q VALUE = strongest learned behavior.
  • MIN Q VALUE = worst learned behavior.
  • Hover graph points to see exact X/Y values.

WHY THERE ARE MANY GRAPH POINTS

The game has around 27 possible states from combinations of:

  • 3 X-directions
  • 3 Y-directions
  • 3 distance ranges

But every state stores multiple actions:

  • LEFT
  • RIGHT
  • FORWARD
  • BACKWARD
  • SHOOT

So graph points = STATES × ACTIONS. Example: 27 states × 5 actions = 135 possible Q-values.

Q-TABLE MEMORY

The AI memory is stored locally inside your browser using localStorage. Refreshing the page keeps the learned memory unless RESET AI is pressed.

NOTES

  • No backend or server is used.
  • The AI learns progressively over time.
  • Learning only exists on your browser/device.
  • The graph updates live while open.
